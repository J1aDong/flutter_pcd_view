## Context

当前点云查看器已实现基础加载和性能优化（去重、体素下采样、最大点数限制）。现在需要增加高级处理功能以支持更复杂的点云可视化场景：

- **激光雷达扫描线可视化**：需要将离散点按扫描顺序或空间邻近性连接成线
- **数据质量优化**：原始点云常包含噪声和离群点，需要过滤以提高显示质量
- **多源数据融合**：多个点云文件需要合并显示，如多角度扫描结果

## Goals / Non-Goals

**Goals:**
- 点连线功能支持两种模式：近邻自动连接（KD-Tree）和顺序连接
- SOR（统计离群点去除）和 ROR（半径离群点去除）两种杂点去除算法
- 多点云合并显示，支持独立颜色标记

**Non-Goals:**
- 点云配准（ICP 算法）- 自动对齐不同坐标系
- 点云分割和分类
- 3D 网格重建

## Decisions

### 1. 点连线算法选择

**决定**: 两种模式并存
- **顺序模式**: 按点在文件中的顺序依次连接，适用于激光雷达扫描线
- **近邻模式**: KD-Tree 近邻搜索，适用于无序点云重建轮廓

**理由**: 不同数据源需要不同连接策略。激光雷达数据天然有序，直接连接效率高；而融合后的点云需要空间搜索。

### 2. 杂点去除算法选择

**决定**: 同时实现 SOR 和 ROR
- **SOR (Statistical Outlier Removal)**: 计算每个点到 k 近邻的平均距离，过滤偏离均值超过 n 倍标准差的点
- **ROR (Radius Outlier Removal)**: 统计指定半径内的邻居数，过滤邻居数低于阈值的点

**理由**: 两种算法适用于不同场景。SOR 适合整体噪声，ROR 适合孤立点。

### 3. 点云融合实现

**决定**: 简单合并 + 可选颜色标记
- 支持多点云输入，直接合并坐标
- 每个点云可选独立颜色，便于区分来源

**理由**: 避免复杂的配准算法，保持简单。如需配准可使用外部工具预处理。

### 4. 模块组织

**决定**: 新增 `rust/src/processing/` 模块，与 `optimization/` 并行
```
rust/src/processing/
├── mod.rs
├── connectivity.rs    # 点连线
├── outlier.rs         # 杂点去除
└── merge.rs           # 点云合并
```

**理由**: 与优化模块职责分离，processing 处理高级变换，optimization 处理性能优化。

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| KD-Tree 构建耗时 | 使用已有的空间索引库或惰性构建 |
| 杂点去除误删有效点 | 提供参数调节（k值、标准差倍数、半径） |
| 线段渲染性能 | 限制最大线段数，使用 GPU 批处理 |
| 多点云内存占用 | 合并前可选择性下采样 |
